# GEMINI Analysis Report

This document provides a comprehensive overview of the Tutochka-Backend project, generated by an AI agent to be used as instructional context for future interactions.

## Project Overview

This is a Kotlin-based REST API for finding public restrooms. It is built using the Ktor framework and uses PostgreSQL with PostGIS for the database. The project follows a clean MVC-like architecture with controllers, services, and repositories. It uses jOOQ for type-safe SQL queries, Koin for dependency injection, and Liquibase for database migrations. The API is documented using OpenAPI and Swagger.

## Building and Running

### Prerequisites

*   Java 17+
*   PostgreSQL 13+ with PostGIS extension
*   Docker (optional)

### Environment Variables

Create a `.env` file in the project root:

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=tutochka
DB_USER=your_username
DB_PASSWORD=your_password
APP_PORT=8080
APP_ENV=development
```

### Commands

*   **Run database migrations:** `./gradlew liquibaseUpdate`
*   **Run the application:** `./gradlew run`
*   **Build the application:** `./gradlew build`
*   **Run unit tests:** `./gradlew test`
*   **Run integration tests:** `./gradlew integrationTest`
*   **Run all tests:** `./gradlew check`
*   **Check code style:** `./gradlew ktlintCheck`
*   **Format code:** `./gradlew ktlintFormat`
*   **Build Docker image:** `./gradlew buildImage`
*   **Run with Docker Compose:** `docker-compose up -d`

## Development Conventions

### Code Style

The project uses ktlint to enforce a consistent code style. The build process is configured to automatically format the code and run style checks.

### Testing

The project has a comprehensive test suite that includes both unit and integration tests. Integration tests use Testcontainers to spin up a real PostgreSQL database, ensuring that the tests are as close to a production environment as possible.

### API Documentation

The API is documented using OpenAPI and Swagger. The Swagger UI is available at `http://localhost:8080/swagger-ui.html` when the application is running.

### Database Migrations

Database schema changes are managed using Liquibase. Migration scripts are located in the `src/main/resources/db/changelog/` directory.
