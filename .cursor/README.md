# Cursor CLI Configuration

Этот файл содержит конфигурацию разрешений для Cursor CLI на уровне проекта.

## Файл конфигурации

`.cursor/cli.json` - настройки разрешений для данного проекта

## Разрешенные операции

### Shell команды
- `git` - все git команды (кроме push и опасных операций)
- `./gradlew` - запуск Gradle wrapper
- `gradle` - запуск Gradle
- `grep` - поиск в файлах
- `find` - поиск файлов
- `cd`, `ls`, `pwd` - навигация по файловой системе
- `cat`, `head`, `tail` - просмотр файлов
- `wc`, `mkdir` - утилиты
- `sed`, `awk` - обработка текста
- `which`, `echo` - системные утилиты

### Операции с файлами
- `Read(*)` - чтение любых файлов проекта
- `Write(*)` - запись в файлы проекта
- `Delete(*)` - удаление файлов (с ограничениями через deny)
- `Search(*)` - поиск по кодовой базе
- `Terminal(*)` - выполнение команд в терминале

## Запрещенные операции

### Опасные Shell команды
- `rm`, `rmdir` - удаление файлов/директорий
- `git push`, `git push --force`, `git push -f` - отправка изменений в удаленный репозиторий
- `git reset --hard` - жесткий сброс изменений
- `git clean -fd` - очистка неотслеживаемых файлов
- `drop`, `truncate`, `dd`, `shred` - опасные операции с данными

### Защита конфиденциальных данных
- `Read(.env)`, `Read(.env.*)` - чтение файлов с переменными окружения
- `Write(.env)`, `Write(.env.*)` - запись в файлы с переменными окружения

## Использованные команды в сессии

Во время работы над проектом использовались следующие команды:

### Git операции
- `git status` - проверка статуса репозитория
- `git log` - просмотр истории коммитов
- `git diff` - просмотр изменений
- `git add` - добавление файлов в индекс
- `git commit` - создание коммитов
- `git restore --staged` - отмена индексации файлов

### Поиск и анализ
- `grep` - поиск паттернов в файлах
- `find` - поиск файлов по имени
- `codebase_search` - семантический поиск по коду

### Работа с файлами
- `read_file` - чтение файлов
- `write` - создание/запись файлов
- `search_replace` - замена текста в файлах
- `list_dir` - просмотр содержимого директорий

### Запуск проекта и тестов
- `./gradlew` - запуск Gradle задач (компиляция, тесты)
- `gradle dependencies` - просмотр зависимостей

### Утилиты
- `cat`, `head`, `tail` - просмотр содержимого файлов
- `wc` - подсчет строк/слов
- `mkdir` - создание директорий
- `sed` - обработка текста

## Безопасность

Конфигурация настроена так, чтобы разрешить все необходимые операции для разработки, но запретить опасные действия:
- Удаление файлов через rm
- Отправка изменений в удаленный репозиторий
- Доступ к файлам с секретами (.env)
- Опасные операции с git (hard reset, force push)

## Примечания

- Для глобальных настроек используйте `~/.cursor/cli-config.json`
- Настройки на уровне проекта имеют приоритет над глобальными
- `deny` список имеет приоритет над `allow` списком
